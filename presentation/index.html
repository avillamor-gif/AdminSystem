Use Quick Launch shortcuts

Jump to any common HR action directly from the dashboard without navigating menus.

Modules 6 – 8

Travel, Office Supplies & Equipment Requests

✈️

Travel Requests

  • Submit a business travel request with destination, dates, and purpose
  • HR/Admin approves or rejects with comments
  • Attach documents (tickets, itinerary)
  • Track status: Pending → Approved → Completed
  • View all your past requests under My Travel Requests
📦

Office Supplies

  • Browse available supply items and current stock levels
  • Submit a supply request for the items you need
  • Admin reviews and fulfils or rejects requests
  • Inventory is updated automatically on fulfilment
  • Admin can add/edit items under Admin → Office Supplies
💻

Equipment / Assets

  • Employees request II assets (laptops, monitors, phones)
  • Admin assigns assets and tracks their condition
  • Each asset has a full lifecycle: Available → Assigned → Returned
  • Maintenance schedules can be recorded
  • Full inventory managed under Admin → Asset Management
🔔 All three request types trigger real-time in-app notifications to the approving admin, and back to the employee on decision.

System Feature

Notifications & Real-Time Updates

How notifications work

1

Event triggers a notification

Any request submission (leave, travel, supplies, equipment, publications, leave credits) creates a notification row in the database.

2

Delivered to the right person

Supervisors and admins are notified of new requests. Employees are notified of decisions (approved / rejected).

3

Bell indicator

The notification bell in the header shows an unread count badge. Clicking it lists all unread notifications with links to the relevant request.

4

Auto-dismissal

Notifications are automatically marked read when the action is taken (e.g., approving a leave request clears the notification).

Notification types

📅

Leave Requests

New requests → supervisors. Decision → employee.

💰

Leave Credits

Credit requests → HR. Approval/rejection → employee.

✈️

Travel Requests

New requests → admins. Decision → requester.

📦

Supply & Equipment

Request → admin stock manager. Fulfilment → employee.

📰

Publications

Publication requests → admin. Decision → requester.

Technical

Technology Stack

Next

Next.js 14 App Router

Server and client components. API routes for secure server-side operations. Middleware-based auth protection on all dashboard routes.

🟢

Supabase (PostgreSQL)

Hosted PostgreSQL with Row-Level Security on every table. Real-time subscriptions for notifications. Supabase Auth for user management. Storage for file uploads.

RQ

React Query

All data fetching goes through custom hooks → service layer → Supabase. Automatic cache invalidation after mutations keeps UI always in sync.

🎨

Tailwind CSS

Utility-first styling with a custom orange brand colour. Reusable UI library: Button, Card, Modal, Badge, Avatar, Input, Select, ConfirmModal.

TS

TypeScript

Strict types throughout. Database types auto-generated from Supabase schema. InsertTables / UpdateTables helpers keep service layer type-safe.

🔐

Role-Based Access

Permissions resolved at runtime from user_roles table. ProtectedRoute guards redirect unauthorised users. Service-role API routes bypass RLS for admin actions.

⚡ Data flow: Component → Hook (React Query) → Service (Supabase client) → PostgreSQL with RLS — consistent across all 16 modules.
